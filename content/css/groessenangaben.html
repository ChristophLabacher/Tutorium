Größenangaben

Pixel

Größen können mit einem absoluten Pixelwerte spezifiziert werden.

div	{
	font-size: 20px;
}

Prozent

Eine weitere Möglichkeit ist es, einen Prozentwert anzugeben. Dieser bezieht sich immer auf die Größe des Elternelements. (Wenn das <div>-Element im oberen Beispiel sich also in einem <header>-Element mit width: 500px befindet, so ist es 250px breit.) Prozentangaben für Größen sind besonders im Responsive-Design sehr häufig.

div	{
	width: 50%;
}

em

Ein em entspricht der Schriftgröße des Elements (diese kann auch vererbt sein). Im oberen Beispiel wäre das <div>-Element also 100px breit. Die Einheit em ist besonders praktisch für die Eigenschaft line-height, die damit relativ zur Schriftgröße angegeben werden kann.

div	{
	font-size: 20px;
	width: 5em;
	margin: 0.5em;
}

rem

Der Gedanke, em als eine Art Grundeinheit eines Dokuments zu verwenden und so ein Raster zu schaffen liegt nahe. Dabei stößt man jedoch schnell auf ein Problem. Möchte man zum Beispiel ein <h1>-Tag formatieren und verändert dessen Schriftgröße, so ist die Rastereinheit für alle weiteren Eigenschaften dieses Tags verändert. Dafür gibt es die Größeneinheit rem (rootem). Ein rem entspricht der für die Dokumentwurzel (also das <html>-Tag) festgelegten Schriftgröße. Im oberen Beispiel bedeutet das, dass das <div>-Element 5 × 15px, also 75px breit ist. Dass die Schriftgröße des Elements verändert ist, hat keinen Einfluss.

html	{
	font-size: 15px;
}	

div	{
	font-size 20px;
	width: 5rem;
	padding: 1rem;
}

auto

Häufig ist es auch möglich anstelle einer Größenangabe den Wert auto anzugeben (in der Regel ist das sogar der Standardwert). Der Browser ermittelt dann selbst eine passende Größe. Gibt man zum Beispiel bei einem Bild eine definitive Höhe, aber als Breite auto an, so wird es proportional skaliert. Gibt man bei einer Box eine Höhe, aber keine Breite an, so passt sich die Breite dem Inhalt, sowie dem verfügbaren Platz an.
