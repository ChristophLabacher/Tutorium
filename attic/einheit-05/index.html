Positionierung

Meistens müssen wir uns um die Positionierung von Block-Elementen keine Gedanken machen: Sie werden nacheinander vom Browser, in der Reihenfolge wie sie im HTML-Dokument angegeben sind, untereinander angezeigt. Diesen Zustand der Positionierung (also ganz normal im Dokumentfluss) nennt man static.

static

div	{
	position: static;
}

Dies ist der Standardwert der Eigenschaft position, wir müssen ihn also fast nie ausdrücklich verwenden. Es gibt noch einige andere Arten der Positionierung:

relative

div	{
	position: relative;
	top: 20px;
	left: -40px;
}

Ist der Wert der Eigenschaft position relative, so wird das Element relativ zu seiner normalen Position (also der, die es hätte wenn der Wert static wäre) verschoben.

Wie weit und in welche Richtung verschoben werden soll, wird durch die Eigenschaften top, bottom, left und right angegeben, wobei immer nur die Verwendung von zwei Sinn ergibt (einer horizontalen und einer vertikalen).

Das <div>-Element im oberen Code Beispiel wäre also von seiner Ausgangsposition aus 20px nach unten und 40px nach links verschoben worden.

Wird die Positionierung relative vorgenommen, so ist wichtig zu beachten, dass das <div>-Element, nur optisch verschoben wird, logisch befindet es sich noch an seiner ursprünglichen Position. Deswegen wird anderer Inhalt auch nur dort verdrängt, an der neuen Position überlagert das Element den anderen Inhalt.

absolute

<div class="container">
	<div class="content">Content</div>
</div>
.container	{
	position: relative;
}

.content	{
	position: absolute;
	bottom: 20px;
	right: 25px;
}

Ist der Wert der Eigenschaft position absolute, so wird das Element relativ zum nächsten Elternelement verschoben, des position-Eigenschaft nicht den Wert static besitzt.

In diesem Code-Beispiel wird das Element mit der Klasse content also, ausgehend von der Position seines Elternelements mit der Klasse container, um 20px nach oben und um 25px nach links verschoben. Wichtig ist, dass das Element mit der Klasse container nicht den Wert static besitzt, sondern relative. Damit das Element aber trotzdem nicht verschoben wird, sind keine Angaben weiteren gemacht: das Element wird von seiner Ursprungsposition um 0px verschoben.

Ein Problem das häufig auftritt, wenn der Wert absolute verwendet wird ist, dass ein Element, das diesen Wert besitzt sich (warum auch immer) nicht mehr auf die Größe seines Elternelements auswirkt. Das Element mit der Klasse container im oberen Beispiel hätte also die Höhe 0px. Durch dieses Problem ist das Verwenden von absolute meistens nicht praktikabel.

fixed

div	{
	position: fixed;
	bottom: 40px;
	right: 25px;
}

Ist der Wert der Eigenschaft position absolute, so wird das Element relativ zum Browserfenster verschoben.

In diesem Code-Beispiel befindet sich das Element dann also in der unteren rechten Ecke des Browserfensters, 40px vom unteren und 25px vom rechten Rand entfernt. Es bleibt an dieser Stelle auch wenn gescrollt wird.

Auch bei diesem Wert wird das Element aus dem Dokumentfluss gehoben und die Größe wirkt sich auf keine andern Elemente mehr aus, in diesem Fall ist es jedoch meistens sinnvoll, da sich das Element ja an einem gänzlich anderen Ort befindet als zuvor.

Dieser Wert ist besonders für Menüs oder Header geeignet, die beim Scrollen im Bild bleiben sollen.

z-index

Durch Verwendung der Eigenschaft position ist es möglich, dass sich verschiedenen Elemente überlagern. Generell gilt: je weiter unten sich ein Element im HTML-Dokument befindet, desto höher liegt es. Mit Hilfe der Eigenschaft z-index lässt sich jedoch darauf Einfluss nehmen: Elemente mit höherem Zahlenwert als Wert der Eigenschaft z-index überlagern Elemente mit einem niedrigeren Wert. z-index darf auch negative Werte enthalten.

div	{
	z-index: 2;
}

Box-Styling

Es gibt verschiedene Eigenschaften, mit denen das Aussehen von Block-Elemente beeinflusst werden kann.

Größe

div	{
	width: 100%;
	height: 500px;
	padding: 30px;
	margin: 50px;
}

Bereits bekannt sind die Eigenschaften für die Größe des Elements (Einheit 3), width (Breite), heigth (Höhe), padding (Innenabstand) und margin (Außenabstand).

Zusätzlich zu definitiven Größenangaben, ist es bei Eigenschaften, die eine Größe als Wert erwarten immer auch möglich auto als Wert anzugeben (dies ist der Standardwert wenn nichts angegeben wird). In diesem Fall wird versucht eine passende Größe zu finden. Gibt man Beispiel bei einem Bild eine definitive Höhe, aber als Breite auto an, so wird es proportional skaliert. Gibt man bei einer Box eine Höhe, aber keine Breite an, so passt sich die Breite dem Inhalt, sowie dem verfügbaren Platz an.

Hintergrundfarbe

div	{
	background-color: rgba(0,0,0,0.5);
}

Ebenfalls bekannt ist die Eigenschaft background-color (Einheit 2), mit der sich (auch halbtransparente) Hintergrundfarben definieren lassen.

Hintergrundbild

div	{
	background-image: url("../imgs/bg.jpg");
	background-repeat: no-repeat;
	background-size: cover;
	background-position: left top;
}

Neu sind die Eigenschaften für Hintergrundbilder.

Mit Hilfe der Eigenschaft background-image wird angegeben, welches Bild als Hintergrundbild verwendet werden soll. Dabei wird als Wert url("relativer Pfad") verwendet. Der Pfad zum Bild ist relativ, ausgehen von der Position des Stylesheets anzugeben.

background-repeat gibt an, ob ein Hintergrundbild gekachelt wird. Mögliche Eigenschaften sind no-repeat, repeat-x (horizontale Kachelung), repeat-y (vertikale Kachelung) und repeat (horizontale und vertikale Kachelung).

Mit background-size wird die Größe des Hintergrundbildes angegeben. Dabei können entweder zwei Größenangaben für Höhe und Breite (zum Beispiel 300px 60px oder 600px auto), oder die Werte cover (das Hintergrundbild wird so skaliert, dass die Box immer abgedeckt ist, auch wenn ein Teil vom Bild abgeschnitten wird) oder contain (das Hintergrundbild wird so skaliert, dass auf jeden Fall das gesamte Bild sichtbar ist, auch wenn ein Teil der Box nicht abgedeckt wird) verwendet werden.

background-position gibt für diese Fälle an, welche Punkt der Box und des Hintergrundbilds als übereinander gelegt werden sollen. Dabei wird immer ein vertikaler (left, center, right) und ein horizontaler Wert (top, center, bottom) erwartet. Bei center center, liegt also der Mittelpunkt des Bildes auf dem Mittelpunkt der Box, bei left top die linke obere Ecke des Bilder auf der linken oberen Ecke der Box, usw.

Transparenz

div	{
	opacity: .5;
}

Die Eigenschaft opacity gibt die Transparenz einer Box (inklusive Inhalt) an. Dabei ist der Wert 0 völlig transparent und der Wert 1 vollkommen opak.

Schatten

div	{
	box-shadow: 3px 3px 4px 5px #000000;
}

box-shadow gibt einer Box einen Schatten. Dabei sind müssen mehrere Werte angegeben werden: horizontale Verschiebung, vertikale Verschiebung, Blur, Größe und Farbe.

Blur und Größe sind optional, folgende Angaben sind also auch möglich:

header	{
	box-shadow: 3px 3px #000000;
}

footer	{
	box-shadow: 3px 3px 4px #000000;
}

Konturen

Wie bereits in Einheit 3 behandelt, sind Konturen von Boxen möglich.

Runde Ecken

div	{
	border-radius: 10px;
}

Der Wert der Eigenschaft border-radius gibt den Radius der abgerundeten Ecken einer Box an. Dabei kann auch für jede Ecke, ausgehend von der links oberen im Uhrzeigersinn ein eigener Wert angegeben werden:

div	{
	border-radius: 10px 5px 10px 5px;
}
