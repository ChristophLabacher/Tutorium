Begriffe

HTML ist eine Auszeichnungssprache, die Strukturierung des Dokuments erfolgt also mit Hilfe von Tags. Es gibt öffnende und schließende Tags. Tags können bestimmte Attribute mit Werten haben, diese werden im öffnenden Tag notiert.

<tag attribut="wert">Inhalt</tag>

Attribute spielten vor allen Dingen in den späten 90ern, als CSS noch nicht verbreitet war, eine große Rolle, da hier das komplette (rudimentäre) Styling der Website durch Attribute erfolgt. Durch CSS nahm die Bedeutung der meisten Attribute enorm ab. In HTML5 dürfen beliebige eigene Attribute eingeführt werden, sie gewinnen also zunehmend wieder an Bedeutung.

Kommentare

Durch seine einfache Struktur braucht ein HTML-Dokument nur selten Kommentare, also vom Browser nicht dargestellten Text. Anders als bei den meisten Sprachen gibt es bei HTML nur eine Art von Kommentaren, die sowohl ein-, als auch mehrzeilig sein kann. Das öffnende und schließende Tag eines Kommentars entspricht nicht der gewohnten HTML-Syntax:

<!-- Auskommentierter Text -->
Achtung: Kommentare können nicht ineinander verschachtelt werden!

DOCTYPE

Über die Jahre wurden verschieden Versionen von HTML entwickelt. Damit der Browser weiß in welcher Version von HTML das Dokument geschrieben ist, wird an oberster Stelle im Dokument, also noch vor di <html>-Tag, das DOCTYPE-Tag eingefügt. Bei früheren Version war das DOCTYPE-Tag furchtbar lang, komplex und unübersichtlich; bei der für uns relevanten, aktuellen Version HTML5 wurde das DOCTYPE-Tag stark gekürzt:

<!DOCTYPE HTML>

Informationen im head-Tag

Im <head>-Tag eines HTML-Dokuments werden verschiedene wichtige Informationen abgelegt, die sich zwar auf das Verhalten des Browsers auswirken, von diesi aber nicht innerhalb der Website angezeigt werden.

Informationen

Im <title>-Tag wird der Titel der Seite angegeben, wie er oben im Browserfenster angezeigt werden soll. Dieses Tag ist laut HTML-Spezifikation zwingend erforderlich.

Das <base>-Tag wird selten verwendet und gibt die Basis-URL der Website an, das bedeutet die URL, von der aus alle relativ angegeben Pfade im Dokument ausgehen.

Es gibt noch einige weitere Informations-Tags, diese sind aber veraltet und wurden weitgehend durch die <meta>-Tags ersetzt.

meta-Tags

Die meisten Informationen über den Inhalt der Website, die für Suchmaschinen relevant sind werden über <meta>-Tags in das Dokument geschrieben:

<meta name="author" content="Name des Autors">
<meta name="description" content="Kurze Beschreibung des Inhalts der Website">
<meta name="keywords" content="Wichtige Stichwörter/Suchbegriffe zum Inhalt der Website">
<meta name="date" content="Erstellungsdatum">
<meta http-equiv="language" content="deutsch, de">

<meta>-Tags sind nicht zwingend in eini Dokument erforderlich, folgendes Tag sollte aber generell in jeder Seite auftauchen:

<meta charset="utf-8">

Damit wird di Browser mitgeteilt was für ein Zeichensatz die HTML-Datei verwendet; das ist bei der Verwendung von Sonderzeichen und insbesondere Umlauten relevant. Mit UTF-8 ist man generell auf der sicheren Seite (Achtung, die Datei muss auch tatsächlich in diesi Format codiert sein!).

Eingebundene Dateien

Mit di <link>-Tag können andere Dateien in das Dokument eingebunden werden. Es wird am häufigsten für CSS-Dateien benutzt, kann aber zum Beispiel auch das Icon der Website einbinden:

<link rel="shortcut icon" href="/favicon.ico" />

Zum Einbinden von Javascript-Dateien wird das <script>-Tag verwendet.

Die genaue Syntax zum einbinden von CSS und Javascript wird zu gegebenen Zeitpunkt besprochen.

Beispiel eines üblichen Dokument-Kopfes

Texte & Typografie

Da HTML für das Strukturieren digitaler wissenschaftlicher Dokumente entwickelt wurde, gibt es viele Tags, die eine genaue Textstrukturierung erlauben. Teilweise wirken sich Tags nicht auf das Aussehen eines Textes aus, können aber im Dokument Sinn ergeben, zum Beispiel um auf eine bestimmte Art des Inhalts hinzuweisen.

Überschriften

Es können Überschriften ersten bis sechsten Grades ausgezeichnet werden. Die Tags hierfür heißen, abgeleitet von „heading“, <h1> bis <h6>.

<h1>Überschrift ersten Grades</h1>
<h2>Überschrift zweiten Grades</h2>
<h3>Überschrift dritten Grades</h3>
<h4>Überschrift vierten Grades</h4>
<h5>Überschrift fünften Grades</h5>
<h6>Überschrift sechsten Grades</h6>

Absätze

Ein Absatz (paragraph) von Text wird durch das p-Tag gekennzeichnet. Üblicherweise folgt auf das <p>-Tag ein Abstand.

<p>Ein Absatz eines längeren Textes</p>

Ein Zeilenumbruch (linebreak) innerhalb eines Absatzes wird durch das <br>-Tag erzeugt, sollte aber möglichst sparsam eingesetzt werden.

Ein Zeilenumbruch erreicht man durch ein einfaches Tag.<br>
Es besitzt kein schließendes Pendant.

Textauszeichnung

Diese Tags können innerhalb eines Textabschnitts/einer Textzeile verwendet werden. Teilweise erreichen verschiedene Tags visuell das gleiche Ergebnis, um eine möglichst gute Semantik des Dokumentes zu erreichen sollten jedoch vom Kontext abhängig bestimmte Tags verwendet werden.

Folgende Tags ändern die Darstellung von Text, der sich vom restlichen Text absetzt, aber keine spezielle Betonung oder Wichtigkeit besitzt. Sie sollen verwendet werden wenn eine rein optische Veränderung erwünscht ist.

Das b-Tag markiert Text, der fettgedruckt (bold) dargestellt wird.

Das i-Tag markiert Text, der üblicherweise kursiv (italic) dargestellt wird, beispielsweise ein Fremdwort, ein Fachbegriff oder ein Name.

Das u-Tag markiert Text, der unterstrichen (underlined) wird.

Das s-Tag markiert Text, der durchgestrichen (strikethrough) ist.

Das small-Tag wird für kleiner geschriebenen Text verwendet.

Folgende Tags sollten im Falle einer spezielle Betonung oder Wichtigkeit verwendet werden. Sie zeigen eine logische Veränderung im Kontext des Textes an.

Das em-Tag (emphasis) wird üblicherweise kursiv dargestellt und steht für eine besondere Betonung (beispielsweise eines einzelnen Wortes in eini Satz).

Das strong-Tag wird standardmäßig fettgedruckt dargestellt und markiert Text, der besonders hervorgehoben werden soll, zum Beispiel eine Warnung.

Während es keinen Sinn hat, das Aussehen der rein optischen Tags später mit CSS zu verändern (b für bold sollte immer fettgedruckt bleiben), kann es bei den logischen Tags durchaus angiessen sein (strong könnte auch als roter Text dargestellt werden).

Einige Tags müssen nicht zwingend eine optische Veränderung bewirken, sondern können auch nur die Art des Inhalts spezifizieren:

Das time-Tag markiert eine Uhrzeit (im 24-Stunden-Format).

Das blockquote-Tag markiert einen Textabschnitt, der ein Zitat ist. Das Pendant das innerhalb eines Absatzes verwendet werden soll ist das q-Tag. Mit di cite-Tag sollte die dazugehörige Quelle markiert werden.

Das code-Tag markiert einen Code-Block (dieser wird meist in einer Monospace-Font dargestellt). Achtung: Tags innerhalb dieses Tags werden trotzdi als HTML-Interpretiert!

Listen

Listen spielen in HTML eine große Rolle: Sie werden nicht nur für Aufzählungen, sondern häufig auch für Menüstrukturen o. ä. eingesetzt.

Ein Listenelient wird immer durch das li-Tag (list) markiert. Um was für eine Art von Liste es sich handelt ergibt sich aus di umschließenden Tag: ul (unordered list) für Listen mit einfachen Aufzählungszeichen und ol (ordered list) für nummerierte Listen.

<h3>Unsortierte Liste</h3>
<ul>
	<li>Listenelement</li>
	<li>Listenelement</li>
	<li>Listenelement</li>
</ul>

<h3>Sortierte Liste</h3>
<ol>
	<li>Listenelement</li>
	<li>Listenelement</li>
	<li>Listenelement</li>
</ol>

Unsortierte Liste

  • Listenelement
  • Listenelement
  • Listenelement

Nummerierte Liste

  1. Listenelement
  2. Listenelement
  3. Listenelement

Definitionslisten

Eine besondere Liste ist die Definitionsliste. Sie wird durch das dl-Tag (definition list) erzeugt. In ihr befinden sich ein oder mehrere dt-Tags (defintion term), die einen zu erklärenden Text enthalten. Auf jedes dt-Tag müssen ein oder mehrere dd-Tags (defintion) folgen, dass die zugehörige Erklärung enthalten.

<h3>Defintionsliste</h3>
<dl>
	<dt>li-Tag</dt>
	<dd>Listenelement</dd>
	<dt>ul-Tag</dt>
	<dd>Unsortierte Liste</dl>
	<dt>ol-Tag</dt>
	<dd>Nummerierte Liste</dd>
</dl>
